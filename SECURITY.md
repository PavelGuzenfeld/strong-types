# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest release | Yes |
| Previous minor | Best effort |
| Older | No |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

Please report security vulnerabilities by [opening a GitHub issue](https://github.com/PavelGuzenfeld/strong-types/issues/new?labels=security&title=Security%3A+).

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: within 48 hours
- **Initial assessment**: within 1 week
- **Fix or mitigation**: best effort, typically within 2 weeks for critical issues

## Scope

This policy covers:
- The C++ header-only library in `include/`
- The test and fuzz harnesses
- The CI/CD workflows in `.github/workflows/`
